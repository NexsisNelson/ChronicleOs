# Getting Started with ChronicleOS

## 5-Minute Quickstart

### 1. Clone & Install

```bash
git clone <repo-url>
cd chronicle-os
npm install
cd apps/agents && python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
cd ../..
```

### 2. Copy Environment Templates

```bash
cp apps/agents/.env.example apps/agents/.env
cp apps/dashboard/.env.example apps/dashboard/.env.local
```

### 3. Start Services

```bash
# Terminal 1: Dashboard
npm --workspace=apps/dashboard run dev
# Opens http://localhost:3000

# Terminal 2: Agents
cd apps/agents
python main.py --task "Analyze the latest trends in decentralized storage"
```

## What You're Looking At

### The Dashboard
Open **http://localhost:3000** to see:
- **Overview**: Live agent status and session info
- **Memory Timeline**: Agent state changes over time (will populate once agents run)
- **Artifact Explorer**: Files generated by agents
- **Agent Monitor**: Real-time logs and metrics

### The Agents
Running `python main.py` executes a 3-agent workflow:

```
Researcher Agent (30s)
  ↓ gathers raw data
Architect Agent (20s)
  ↓ synthesizes artifacts
Auditor Agent (10s)
  ↓ reviews quality → ✅ APPROVED
```

All three run locally with placeholder data for now.

## Key Concepts

### CID (Content Identifier)
A unique hash of a file stored on Walrus. Example: `z1A2B3C4D5E6F7G8H9I0`

Use CIDs to reference artifacts without storing them locally.

### MemWal Proof
A cryptographic signature proving an agent made a particular decision. Enables auditability and verification.

### Shared Workspace
A standardized directory structure where agents collaborate. The Researcher saves findings → Architect reads findings → Auditor verifies.

## Troubleshooting

### "Connection refused" when starting agents
- Agents don't need external services for Phase 1
- This error shouldn't happen with placeholder data
- Check your Python venv is activated

### Dashboard shows no data
- Run the agents first: `python apps/agents/main.py --task "..."`
- Agents populate the dashboard in real-time
- Phase 2 will add Walrus integration for persistent data

### Can't import `memwal_adapter`
- Install the package: `pip install -e packages/memwal-adapter`
- Ensure you're in the Python venv

## API Reference

### Running an Agent Workflow

```bash
python main.py \
  --task "Your research task" \
  --session-id "my_session_123" \
  --debug
```

### MemWal Adapter

```python
from memwal_adapter import MemWalChatMessageHistory

memory = MemWalChatMessageHistory(
    endpoint="http://localhost:8000",
    session_id="session_1",
    agent_id="researcher"
)

# Use with LangChain
from langchain.memory import ConversationBufferMemory
lc_memory = ConversationBufferMemory(chat_memory=memory)
```
